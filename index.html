问题

长上下文 Agent 首 token 延迟高、显存吃紧，外挂记忆质量参差。要在几乎不损失准确率的前提下压缩上下文，并量化整条链路的性能与崩溃边界。

思路

用轻量 4B 模型对每个子句算困惑度相关性来做压缩，而非粗暴截断；再搭一套自动评测框架，把"压缩前后"和"不同并发"的表现量化出来。

我做了什么
  • 压缩算法：独立设计 query-aware 子句级压缩，经 token / 句子 / 子句三粒度对照选定子句级；用 4B sidecar 困惑度打分替代 30B 主模型，可并行可缓存。
  • 评测框架：从 0 搭建 LongMemEval 全链路自动评测；主导跨会话检索消融，Dense+BM25+RRF 使端到端准确率 0.20→0.40（翻倍），recall@150 = 0.96。
  • 系统集成与调优：注册表式五压缩器热插拔；昇腾 NPU 上定量分析 KV cache 卸载 vs 压缩的带宽 / 延迟权衡，定位 C16 并发的 OOM 崩溃根因。

一个"记得住、做得完、不闯祸"的 AI 伙伴，以分层记忆、自证验证闭环、审批沙箱为三大内核。

问题

多数 Agent 把历史硬塞进窗口——一满就遗忘、自相矛盾；声称"已完成"却没有验证；拿到危险权限就直接执行。

思路

把"记忆"做成分层库存、把"完成"变成可证明的收敛、把"危险操作"关进审批与沙箱——从机制上而非提示词上解决三大顽疾。

我做了什么
  • 分层记忆 + 时态知识图谱：L0 偏好卡 → L5 向量索引；自研冲突消解——同一事实前后矛盾时不删除、只标记失效，默认召回当前有效版本；用户 / 项目双层，经验跨项目沉淀。
  • 上下文编排器：全局 token 预算 + 跨层去重（按权威性保留一份）+ 超预算只丢弃不截断，产出可观测的 BudgetReport。
  • 验证闭环 + 审批沙箱：三态收敛判定 + 哈希防篡改 TDD；危险操作触发三态审批，文件变更经 Workspace 沙箱与黑名单拦截。

TeacherMaxAI — AI 教学智能体平台

把大模型深度融入课程、考试、教案、问答的智慧校园后端，用任务图调度与增强 RAG 解决真实教学场景问题。

问题

教学知识库初版向量检索几乎全部失败；用户一句话指令常含多个相互依赖的子任务，难以高效执行。

思路

检索端用多策略并行召回 + RRF 融合重排把准确率救起来；调度端把指令拆成任务图（DAG），有依赖的串行、无依赖的并行。

我做了什么
  • 增强型 RAG：语义 + 关键词 + 混合三策略并行召回 → 加权 RRF 融合重排 → 相关度阈值过滤，配 Embedding LRU 缓存（命中 80%+），把不可用的检索重构为可用，并以 A/B 框架量化验证。
  • DAG 任务调度：LLM 意图拆分 + 依赖分析 → 任务图同层并行、跨层串行，延迟队列重试未满足依赖的任务。
  • 统一记忆管理器：以对话轮为检索单元，BM25 回溯历史降低幻觉；按用户加 ReentrantLock + 存储双写保证一致性。
  • 智能文档处理：1177 行类型感知分割——PDF 软换行、代码按函数边界、Markdown 保护代码块、Token 级二分切分。

问题

中医药治疗艾滋病的临床研究，病例数据分散在表格里，靠人工 Excel 做疗效统计低效、易错，也难以做规范的统计学显著性判断。

思路

做一个"录入 + 导入 + 统计"一体的系统：把临床指标结构化存储，按多维度自动汇总分布，并内置规范的疗效评估与统计检验。

我做了什么
  • 数据建模与导入：设计病例信息（PatientFVR / PatientAGR）、知情同意等表结构（DDL），支持 Excel 模板批量导入。
  • 多维度统计：按年龄 / 性别 / 传播途径 / 诊断 / 证型 / 体质 / 治疗方案 / 疗程 / 疗效等 15+ 维度自动分布统计；追踪 CD4、病毒载量、体重、淋巴细胞等治疗前后变化。
  • 疗效评估 + 统计检验：自研疗效指数 (前−后)/前 做有效 / 稳定 / 无效三分类；实现配对样本 t 检验（t / p 值）与卡方检验，对疗效做统计显著性判断。
